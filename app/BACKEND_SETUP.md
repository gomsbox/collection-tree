# 백엔드(실시간 공유) 연동 가이드

이 문서대로 하면 "나의 수집 자랑하기"가 이 브라우저에만 저장되는 로컬 모드에서 벗어나,
누구든 사이트 주소만 알면 **같은 나무를 실시간으로 함께 보고 등록**할 수 있는 모드로 바뀝니다.
비용은 무료(개인 Google 계정 기준)이며, 별도 서버 없이 Google Sheets + Apps Script만으로 동작합니다.

설정하지 않아도 앱은 정상 동작합니다(이 브라우저에만 저장되는 로컬 모드). 필요할 때 아래 순서를 따라 하면 됩니다.

## 1. Google Sheets 준비
1. [sheets.new](https://sheets.new) 로 새 스프레드시트를 만듭니다. 이름은 자유롭게 정하세요(예: "나의 수집 자랑하기 DB").
2. 상단 메뉴에서 **확장 프로그램 → Apps Script** 를 클릭합니다. 새 탭에서 스크립트 편집기가 열립니다.

## 2. 백엔드 코드 붙여넣기
1. 편집기에 기본으로 있는 `myFunction() {...}` 코드를 전부 지웁니다.
2. 이 저장소의 [`apps-script/Code.gs`](../apps-script/Code.gs) 파일 내용을 통째로 복사해서 붙여넣습니다.
3. 좌측 상단 프로젝트 이름을 원하는 대로 바꾸고(예: "collection-tree-backend"), `Ctrl+S`(Mac: `Cmd+S`)로 저장합니다.

## 3. (선택) 시트 초기화 확인
1. 상단 함수 선택 드롭다운에서 `setup`을 고르고 ▶ 실행 버튼을 누릅니다.
2. 처음 실행하면 "권한 검토 필요" 팝업이 뜹니다 → 본인 Google 계정 선택 → "고급" → "(프로젝트 이름)(으)로 이동" → 허용.
   (내가 만든 스크립트이므로 안전합니다. Google이 미검증 앱이라는 경고를 보여주는 것뿐입니다.)
3. 실행이 끝나면 원래 스프레드시트로 돌아가 하단에 `Records`, `Recorders` 시트 탭이 자동으로 생긴 것을 확인합니다.
   (이 단계를 건너뛰어도 배포 후 첫 요청 시 자동으로 만들어집니다.)

## 4. 웹 앱으로 배포
1. 편집기 우측 상단 **배포 → 새 배포**를 클릭합니다.
2. 톱니바퀴(⚙️) 아이콘 → **웹 앱** 선택.
3. 다음과 같이 설정합니다.
   - **설명**: 아무거나(예: "v1")
   - **다음 사용자 인증 정보로 실행**: **나**
   - **액세스 권한이 있는 사용자**: **모든 사용자**
4. **배포** 클릭 → 다시 한번 권한 승인 진행 → 완료되면 **웹 앱 URL**이 표시됩니다.
   `https://script.google.com/macros/s/AKfycb.../exec` 형태의 주소를 복사해 둡니다.

> ⚠️ 액세스 권한을 "모든 사용자"로 하지 않으면 다른 기록자가 로그인/등록할 때 오류가 납니다.
> 이 웹 앱은 시트 소유자(나) 권한으로 실행되므로, 방문자가 Google 계정에 로그인할 필요는 없습니다.

## 5. 앱에 연결
1. `app/config.js` 파일을 엽니다.
2. `SHEET_API_URL` 값에 4번에서 복사한 URL을 붙여넣습니다.
   ```js
   window.CT_CONFIG = {
     SHEET_API_URL: "https://script.google.com/macros/s/AKfycb.../exec",
   };
   ```
3. 저장 후 앱을 새로고침하면 좌측 상단에 **"☁️ 실시간 공유 모드"** 배지가 뜨면서 자동으로 연결됩니다.
   (연결에 실패하면 자동으로 로컬 모드로 전환되고 안내 토스트가 뜹니다 — `SHEET_API_URL`을 다시 확인하세요.)

## 6. 배포 후 확인해볼 것
- 서로 다른 브라우저(또는 시크릿 창)에서 같은 주소로 접속했을 때, 한쪽에서 등록한 키링이 다른 쪽에서도 보이는지
- 이름+비밀번호로 로그인 후 등록 → 로그아웃 → 같은 이름으로 다시 로그인했을 때 내 항목에 앞/뒤/삭제 버튼이 보이는지
- 다른 이름으로 로그인했을 때 남의 항목에는 🔒만 보이고 수정 버튼이 없는지
- Google Sheets를 열어보면 `Records`/`Recorders` 시트에 실제로 행이 쌓이는지

## 코드를 수정하고 다시 배포할 때
그냥 "새 배포"를 또 만들면 **URL이 바뀝니다.** 기존 URL을 유지하려면:
1. Apps Script 편집기에서 코드 수정 후 저장
2. **배포 → 배포 관리** → 기존 배포 옆 연필(✏️) 아이콘 클릭
3. 버전을 **"새 버전"**으로 선택 → **배포**

## 알아두면 좋은 점
- **사진**: Google Sheets 셀 용량 제한(약 5만 자) 때문에 앱에서 첨부 사진을 자동으로 320px 이하로 축소·압축해서 저장합니다. 원본 고화질 보관용은 아닙니다.
- **비밀번호**: 평문으로 저장되지 않고 salt + SHA-256 해시로만 `Recorders` 시트에 저장됩니다. 다만 시트 자체에 접근 권한이 있는 사람(소유자)은 해시값은 볼 수 있으니, 다른 서비스와 같은 비밀번호를 재사용하지 않는 걸 권장합니다.
- **무료 사용량**: 개인 Google 계정 기준 Apps Script 실행 할당량이 있습니다(하루 URL Fetch/실행 횟수 제한). 가족·지인 규모의 사용에는 충분합니다.
- **되돌리기**: 언제든 `app/config.js`의 `SHEET_API_URL`을 다시 빈 문자열로 바꾸면 로컬 모드로 돌아갑니다(기존에 로컬에 저장돼 있던 데이터가 있다면 그대로 남아있습니다).
