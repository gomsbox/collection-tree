# 나의 수집 자랑하기 — 요구정의서

> 작성일: 2026-07-02
> 수정일: 2026-07-03

---

## 1. 개요

**한 줄 정의**: 내가 가진 수집품 중 가장 자랑하고 싶은 것을 캐릭터로 만들어 나무에 매달아가며, 무엇인지 소개하고 제품 링크까지 남겨 나만의 "수집 자랑 트리"를 완성하는 웹 서비스.

수집 자랑 기록을 나무에 캐릭터가 매달리는 형태로 시각화한다. 나무에 달린 캐릭터를 클릭하면 등록해둔 제품 링크로 이동해 실제 제품을 바로 확인할 수 있다. 기록이 쌓일수록 나무가 캐릭터로 채워지는 성취감·과시욕을 자극하는 것이 핵심 재미 요소다.

---

## 2. 서비스 컨셉 정의

수집품은 이름이 무한하다(예: "나이키 운동화", "무선 이어폰", "피규어" 등 끝이 없음). 수집품마다 전용 3D 모델을 만드는 것은 비현실적이므로, **카테고리 단위의 아이콘은 물건을 분류하는 대(大) 카테고리 기준으로 통일**한다. 세부 상품명이 아니라 패션/전자기기/생활용품/뷰티/취미 등 대분류 수준의 아이콘 세트만 정의하고, 사용자는 모달에서 캐릭터 외형과 이 대카테고리 아이콘을 선택해 조합한다.

이렇게 하면 "채워지는 재미"는 유지하면서도, 무한한 수집품 종류를 유한한 대카테고리 아이콘 세트로 현실적인 개발 리소스 안에서 감당할 수 있다.

## 3. 기능 요구사항

### 3.1 MVP (최소 기능 제품)

| # | 기능 | 설명 |
|---|---|---|
| F1 | 기록자 이름 등록/변경 | 최초 진입 시 이름 입력, 이후 변경 가능 |
| F2 | 수집품 등록 | "자랑하기" 클릭 → 모달에서 캐릭터 설정 + 상품 카테고리(대분류) 선택 → 나무에 캐릭터로 매달림 |
| F3 | 캐릭터/카테고리 선택 모달 | 캐릭터 외형 옵션과 대카테고리 아이콘을 함께 선택 → 적용 |
| F4 | 나무 시각화 | 나무에 캐릭터가 등록 순서대로 매달리는 애니메이션, 다 차면 다음 가지로 |
| F5 | 수집품 설명 입력 | 무엇인지 간단히 설명하는 텍스트 입력 |
| F6 | 제품 링크 등록 및 이동 | 등록 시 제품 링크 입력, 나무의 캐릭터 클릭 시 해당 링크가 새 탭에서 열림(출력) |
| F7 | 기록 리스트 | 좌측 패널에 등록된 수집품 목록, 순서 조정(앞/뒤), 삭제 |
| F8 | 개수 카운터 | 현재까지 등록한 수집품 개수 표시 |
| F9 | 저장/불러오기 | Google Sheets에 저장, 재접속 시 불러오기 |
| F10 | 공유 링크 | 내 기록을 다른 사람이 볼 수 있는 링크 제공(자랑 목적 핵심 기능) |
| F11 | 사진 첨부 | 수집품 등록 시 사진 업로드(선택), 있으면 카드/캐릭터에 표시 |

### 3.2 차후 확장 (Nice to have)

| # | 기능 | 설명 |
|---|---|---|
| E1 | 스크린샷 캡처 | 현재 화면을 이미지로 저장 |
| E2 | 배경음악 토글 | 잔잔한 BGM on/off |
| E3 | 카메라 리셋 | 시점 초기화 |
| E4 | 가격/구매처/후기 기록 | 수집품별 부가 정보(설문 결과: 이번 범위에서는 제외, 향후 확장 시 고려) |
| E5 | 카테고리 필터/통계 | 대카테고리별 소비 개수·비중을 별도 화면에서 확인 |
| E6 | 다중 사용자 갤러리 | 여러 기록자의 나무를 한곳에서 둘러보기 |

> 참고: 설문 결과 "가격/구매처/후기"는 이번 범위에서 **제외**하고 "이름+카테고리+제품 링크(+사진)"만 기록하는 심플한 구성으로 확정했다. 추후 필요 시 E4로 확장 가능하도록 데이터 모델에 여유를 둔다.

## 4. 화면/인터랙션 요구사항

### 4.1 메인 화면 레이아웃
- 좌측: 컨트롤 패널(고정, 스크롤 가능) — 타이틀, 기록자 표시, "자랑하기" 버튼, 카운터, 불러오기/비우기, 등록된 수집품 목록
- 중앙~우측: 3D 나무 공간, 캐릭터가 등록 순서대로 나무에 매달림
- 우측 상단: 부가 기능 버튼 그룹(리셋/캡처/음악 — MVP 이후)
- 하단: 카테고리 기준(마스터 데이터) 출처 표기 영역

### 4.2 수집품 등록 플로우
1. "자랑하기" 버튼 클릭 → 모달 오픈
2. 모달에서 캐릭터 외형을 설정하고, 상품 카테고리(대분류) 아이콘을 선택
3. 무엇인지 간단히 설명하는 텍스트를 입력
4. 해당 제품 링크를 입력
5. "적용" 클릭 → 나무의 다음 빈 가지에 캐릭터가 매달리는 애니메이션 재생

### 4.3 기록 리스트 인터랙션
- 각 항목: 카테고리 색상 점 + 수집품 이름 + "앞" / "뒤" (나무 위 겹침 순서 변경) + "x"(삭제)

### 4.4 캐릭터 클릭 인터랙션
- 나무에 매달린 캐릭터를 클릭하면 등록된 제품 링크가 새 탭에서 열린다.

### 4.5 공유 플로우
- "공유 링크 복사" 버튼 → 읽기 전용 뷰 URL 생성(예: `?user=기록자이름` 또는 고유 ID 기반) → 타인은 등록 불가, 조회 및 캐릭터 클릭(링크 이동)만 가능

## 6. 비기능 요구사항

- **정적 호스팅**: GitHub Pages 등 정적 호스팅으로 배포 가능한 구조 지향
- **백엔드**: 별도 서버 없이 Google Sheets + Apps Script(Web App)를 간이 API로 사용
- **반응형**: 모바일에서도 등록/조회가 가능해야 함
- **성능**: 3D 렌더링 라이브러리(Three.js 추정) 사용 시 수집품 개수가 많아져도 프레임 저하 없이 동작해야 함
- **외부 링크 보안**: 제품 링크로 이동 시 `target="_blank"` + `rel="noopener noreferrer"` 적용
- **브라우저 호환**: 최신 Chrome/Safari/Edge 기준

## 7. 기술 스택 제안

- 프론트엔드: Three.js(3D 렌더링) + Vanilla JS 또는 경량 프레임워크
- 데이터 저장: Google Sheets + Google Apps Script (Web App으로 배포해 fetch 기반 CRUD)
- 배포: GitHub Pages (무료, 정적 호스팅)
- 이미지 첨부(F11) 처리 시 주의사항: **Google Sheets는 이미지 저장에 적합하지 않음** → Google Drive API, 또는 Imgur/Cloudinary 등 이미지 호스팅 서비스와 연동해 URL만 시트에 저장하는 구조 필요 (8장 하단 리스크 참고)

## 8. 리스크 및 열린 질문 (Open Questions)

결정이 필요한 항목들을 정리했다. 개발 착수 전에 확정 권장.

1. **대카테고리 마스터 목록**: 몇 개의 대카테고리를 둘지, 카테고리별 기본 아이콘/컬러를 어떻게 정할지
2. **캐릭터 외형 옵션 세트**: 캐릭터 설정 모달에서 제공할 외형 옵션의 종류와 개수
3. **사진 저장소**: Google Sheets는 이미지 바이너리 저장에 부적합 → 별도 이미지 호스팅 연동 방식 결정 필요 (Google Drive / Cloudinary / imgur 등)
4. **제품 링크 유효성**: 깨진 링크·악성 링크 방지를 위한 최소한의 검증(URL 형식 체크 등) 필요 여부
5. **나무 3D 에셋**: 나무 배경 3D 모델을 직접 제작할지, 기존 무료 에셋을 활용할지
6. **공유 링크의 인증 수준**: 링크만 알면 누구나 조회 가능하게 할지, 최소한의 접근 제한(예: 기록자별 URL slug)을 둘지
7. **모바일 대응 범위**: 등록까지 모바일에서 지원할지, 조회만 지원할지

## 10. 개발 진행 현황 (2026-07-03)

MVP(F1~F11)를 정적 웹앱으로 구현 완료했다. `img/` 폴더의 PNG/WEBP/SVG 핫스팟 자료(`CLAUDE_CODE_TASK.md`, `specs/*.json`)를 화면 구성·인터랙션·데이터 모델 기준으로 삼되, 이미지 자체는 AI 목업 참고 자료이므로 그대로 잘라 쓰지 않고 SVG/CSS/JS로 새로 구현했다(패키지의 `README.md`/`CLAUDE_CODE_TASK.md`에 명시된 방침과 동일). `나의-수집-자랑하기` 폴더(기존 별도 프로젝트)는 사용하지 않았다.

### 구현 위치
- `app/` — 정적 사이트 전체 (빌드 도구 불필요, `index.html`을 브라우저로 바로 열거나 아무 정적 서버로 서빙하면 동작)
  - `index.html` — 레이아웃(좌측 컨트롤 패널 + 우측 나무 스테이지 + 하단 카테고리 범례), 등록 모달, 온보딩 오버레이
  - `style.css` — 전체 스타일, 반응형(모바일 1컬럼), 애니메이션(드롭인/흔들림/컨페티 등)
  - `js/data.js` — 대카테고리 8종 / 캐릭터 외형 8종 / 업적 마스터 데이터 (img/specs/interactive-elements.json 기준)
  - `js/storage.js` — 저장/불러오기 어댑터. 현재는 `LocalStorageAdapter`이며, 추후 Google Sheets + Apps Script 연동 시 동일 인터페이스(`loadState/saveState/clearState`)를 유지한 채 어댑터만 교체하면 됨
  - `js/share.js` — 공유 링크(F10) 인코딩/디코딩. 백엔드 없이도 동작하도록 상태를 URL 쿼리(`?view=...`)에 직렬화
  - `js/tree.js` — 나무·캐릭터를 순수 SVG로 렌더링(캐노피/가지/캐릭터 참). 등록 순서 = 렌더 순서이므로 리스트의 "앞/뒤" 조작이 곧 겹침 순서(z-order)가 됨. 화면 캡처(E1)도 SVG 직렬화 → 캔버스 변환 방식으로 외부 라이브러리 없이 구현
  - `js/modal.js` — 등록 모달(캐릭터/카테고리 선택, 설명·링크 입력, 사진 첨부, 실시간 미리보기, URL 유효성 검증)
  - `js/main.js` — 앱 상태 관리, 렌더링, 이벤트 바인딩, 업적/컨페티, 카메라 팬·줌(E3), BGM 토글(E2, Web Audio로 합성한 앰비언트 톤 — 외부 음원 파일 없이 구현)

### 요구사항 커버리지
- F1~F11(MVP) 전부 구현
- E1(캡처), E2(배경음악 토글), E3(카메라 리셋) 구현. E4(가격/구매처/후기)는 요구정의서 방침대로 범위 제외, E5는 카테고리 요약 캐러셀/범례 수준으로 축소 반영, E6(다중 사용자 갤러리)은 백엔드가 필요해 이번 범위에서 제외
- 8장 열린 질문 중 "모바일 대응 범위"는 6장 비기능요구사항("모바일에서도 등록/조회가 가능해야 함")에 맞춰 등록까지 모바일에서 지원하는 것으로 확정. 공유 링크 인증 수준은 "링크만 알면 조회 가능"(추가 인증 없음)으로 확정
- 사진 저장소(열린 질문 3): 현재는 Data URL로 브라우저에만 저장되며 공유 링크에는 포함하지 않음(용량 문제). Google Drive/Cloudinary 등 외부 이미지 호스팅 연동은 추후 과제로 남김

### 실행 방법
```
cd app
python3 -m http.server 8080   # 또는 임의의 정적 서버
# http://localhost:8080 접속
```
`index.html`을 더블클릭해 `file://`로 바로 열어도 동작한다(ES 모듈 미사용, 일반 스크립트 태그 기반).

### 10.1 업데이트 (2026-07-03, 2차)
- 키링 이미지를 이모지 대신 `img/images/png/04_character_options.png`(캐릭터), `img/images/png/03_category_master.png`(카테고리)에서 캐릭터별/카테고리별 영역을 실제로 크롭한 이미지(`app/assets/characters/*.png`, `app/assets/categories/*.png`)로 교체. 나무 키링, 등록 모달의 캐릭터·카테고리 선택 그리드, 미리보기, 리스트 썸네일, 카테고리 캐러셀에 모두 동일한 이미지를 사용해 일관성 확보
- 나무 키링 구성을 "캐릭터 이미지(원형) + 카테고리 아이콘 배지(모서리)" 조합으로 변경, 기존 번호 배지는 카테고리 아이콘으로 대체
- 나무에 매달린 키링과 좌측 리스트 모두 "{기록자}의 {수집품 이름}" 형식의 이름표를 표시
- 키링 클릭 동작을 "새 탭으로 바로 이동"에서 "정보(캐릭터·카테고리·설명·기록자·사진)를 먼저 보여주는 상세 팝업 → 팝업 안의 '제품 보러 가기' 버튼(`target="_blank" rel="noopener noreferrer"`)으로 이동"으로 변경
- 좌측 상단 태그라인을 "나무에 내가 자랑하고 싶은 소비를 키링으로 달아 보세요"로 수정

### 10.2 업데이트 (2026-07-03, 3차) — 간단 로그인 + 다중 기록자 공용 트리
- **간단 로그인(이름+비밀번호)**: `app/js/auth.js` 신설. 이름별 비밀번호를 Web Crypto(SHA-256, salt 포함) 해시로 저장하고, 처음 보는 이름이면 자동으로 계정을 생성한다. 로그인 세션은 `localStorage`에 보관되어 새로고침해도 유지되며, 로그아웃 버튼으로 다른 사람이 로그인할 수 있다. 이 저장소는 "이 브라우저(기기)를 함께 쓰는 사람들" 범위의 보호이며 서로 다른 기기 간 계정 동기화는 되지 않는다는 한계를 명시해 둔다.
- **다중 기록자 공용 트리**: 기존에는 나무 전체가 기록자 1명(`state.recorderName`) 소유였는데, 이제 `records` 배열 하나를 여러 기록자가 함께 채우는 구조로 전환했다. 로그인한 사람은 누구나 자신의 이름으로 키링을 등록할 수 있고, 각 항목은 `record.recorder`로 소유자가 구분된다.
- **권한 체크**: 리스트의 앞/뒤 순서 변경·삭제 버튼은 본인이 등록한 항목에만 노출되고, 남의 항목은 🔒 아이콘으로 표시된다. 함수 레벨에서도 소유자 일치 여부를 다시 검증해 UI를 우회해도 막힌다. "비우기"는 전체 삭제가 아니라 "내 항목 비우기"로 범위를 좁혔다(공용 트리이므로 한 사람이 남의 기록까지 지우지 않도록).
- **내 아이템만 보기**: 등록된 수집 아이템 리스트 상단에 토글을 추가해 전체/내 것만 필터링할 수 있다.
- **구매처 링크 + 간단 후기**: 등록 모달에 "간단 후기(선택)" 입력란을 추가했고, 나무 키링 클릭 시 뜨는 상세 팝업에서 후기와 "구매처 링크로 이동" 버튼을 함께 보여준다.
- **공유 링크(F10)**: 다중 기록자 구조에 맞춰 단일 `recorderName` 필드를 제거하고 항목별 `recorder`만 유지. 공유 URL 인코딩이 이중으로 퍼센트 인코딩되던 기존 버그도 함께 수정해 링크 길이를 줄였다. 읽기 전용 뷰는 로그인 없이 그대로 조회만 가능하다(공유 뷰 자체에 등록 기능을 넣지는 않음).

### 10.3 업데이트 (2026-07-03, 4차) — Google Sheets + Apps Script 백엔드 연동
드디어 8장 열린 질문의 "공유 링크의 인증 수준"과 7장 기술 스택 제안에 있던 백엔드 계획을 실제로 구현했다. 여러 기기·브라우저에서 **같은 사이트 주소로 접속해 실시간으로 같은 나무를 함께** 채울 수 있다.

- **`apps-script/Code.gs`**: Google Sheets를 DB로 쓰는 Apps Script 웹 앱 백엔드. `Records`/`Recorders` 두 시트에 저장하며, `doGet`(목록 조회)·`doPost`(로그인/등록/삭제/순서변경/내 항목 비우기)로 동작. 비밀번호는 salt+SHA-256 해시로만 저장한다. 로그인 성공 시 `token = SHA256(이름+저장된해시)`를 내려주고, 이후 수정 요청마다 이 토큰을 재검증해 다른 사람이 devtools로 recorder 이름만 바꿔 보내도 거부되도록 했다(서버 측 권한 검증, UI 숨김에만 의존하지 않음). 동시 수정 충돌 방지를 위해 `LockService`로 잠금을 건다.
- **`app/config.js`**: 배포한 Apps Script 웹 앱 URL을 붙여넣는 지점. 비워두면(기본값) 기존처럼 로컬 모드로 동작 — 즉 이 기능은 선택 사항이며, 안 써도 앱은 그대로 잘 동작한다.
- **`app/js/remoteStorage.js`**: 위 백엔드와 통신하는 클라이언트. Apps Script의 CORS 프리플라이트 미지원 문제를 피하기 위해 POST 요청은 `Content-Type: text/plain`으로 보낸다. GET 요청은 브라우저 캐시로 다른 사람이 새로 등록한 항목을 놓치지 않도록 매번 캐시를 우회한다.
- **자동 폴백**: 초기 로딩 시 백엔드 연결에 실패하면(URL 미설정, 네트워크 오류 등) 자동으로 로컬 모드로 전환하고 안내 토스트를 띄운다 — 백엔드 설정 실수로 앱이 완전히 먹통이 되지 않는다.
- **공유 링크가 실시간 링크로 단순화**: 백엔드 연동 시 "공유 링크 복사"는 스냅샷 URL 대신 사이트 주소 자체를 복사한다(누구나 그 주소로 들어와 로그인하면 바로 등록도 가능). 백엔드 미설정 시에는 기존 방식(URL에 상태를 담은 읽기 전용 스냅샷)을 그대로 쓴다.
- **사진 첨부 압축**: Google Sheets 셀 용량 제한(약 5만 자)에 맞춰 첨부 사진을 업로드 시 캔버스로 320px 이하로 축소하고 JPEG로 압축한다(로컬 모드에도 동일 적용 — 저장 용량 절약).
- **배포 가이드**: `app/BACKEND_SETUP.md`에 스프레드시트 생성부터 배포, `config.js` 연결까지 단계별로 정리했다. 이 부분은 Google 계정으로 직접 몇 단계를 진행해야 하며, Claude Code가 대신 배포해 줄 수는 없다(브라우저로 Google 계정에 로그인해 실행할 수 있는 주체가 아니기 때문).
- **검증 한계**: 실제 Google Apps Script 배포는 이 환경에서 직접 수행할 수 없어, 클라이언트(`CT.remote`)가 보내는 요청/응답 형식을 `Code.gs`와 동일한 로직의 목(mock) 서버로 재현해 인증·등록·삭제·순서변경·권한거부 시나리오를 모두 검증했다. 실제 배포 후에는 위 "6. 배포 후 확인해볼 것" 체크리스트로 한 번 더 확인하는 것을 권장한다.
